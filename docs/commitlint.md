# Проверка коммитов

Коммиты в проектах обязательно проверяются через githooks с использованием commitlint. Никаких дополнительных действий от разработчика не требуется. При несоответствии формату коммит будет отклонён с кратким сообщением об ошибке.

## Правила коммитов

### Формат заголовка

```
<тип>: <описание>
```

- **Тип** - обязательное поле, указывает на характер изменений
- **Описание** - краткое описание изменений
- После двоеточия **обязательно** идет пробел
- Заголовок пишется с **маленькой буквы**
- В конце заголовка **точка не ставится**
- Оптимальная длина заголовка: **50-72 символа**

### Типы коммитов

| Тип        | Описание                                  | Пример                                     |
| ---------- | ----------------------------------------- | ------------------------------------------ |
| `feat`     | Добавление нового функционала             | `feat: добавить кнопку авторизации`        |
| `fix`      | Исправление ошибок                        | `fix: исправить баг с валидацией email`    |
| `docs`     | Обновление документации                   | `docs: обновить readme файл`               |
| `refactor` | Правки кода без исправления ошибок        | `refactor: добавить мемоизацию компонента` |
| `ci`       | Настройка CI и работа со скриптами        | `ci: добавить переменную окружения`        |
| `build`    | Сборка проекта или изменения зависимостей | `build: обновить версию webpack`           |

## Примеры правильных коммитов

### Коммиты на русском языке

```
feat: добавить в систему UI-компонентов элемент – «кнопка»
fix: добавить правильный цвет для активного состояния обводки инпута
docs: обновить readme файл
refactor: добавить мемоизацию для объекта с карточками услуг
ci: добавить переменную окружения для отправки писем
```

### Коммиты на английском языке

```
feat: add "button" element to the UI component system
fix: add correct color for active input stroke state
docs: update readme file
refactor: add memoization for objects with service cards
ci: add environment variable for sending emails
```

## Проверка работы

```bash
# Неправильный коммит
git commit -m "добавить кнопку"
# ❌ Будет заблокирован

# Правильный коммит
git commit -m "feat: добавить кнопку"
# ✅ Пройдет
```
