# Руководство по работе с видео

В шаблоне используется компонент Video для отображения видео-контента, но с ним есть некоторые особенности.

## Работа с альфа-каналом - прозрачностью фона

Особенностью здесь являются сами движки браузеров и то - как они отображают различные форматы.

| ОС / Браузер | Safari | Chrome                          |
| ------------ | ------ | ------------------------------- |
| **macOS**    | WebKit | Blink (определяется как safari) |
| **iOS**      | WebKit | WebKit\*                        |
| **Windows**  | -      | Blink                           |
| **Android**  | -      | Blink                           |

Особенность macOS / IOS (Webkit): некорректно отображает webm - отображает прозрачность как черный фон <br />
_Решение_: переход на `mp4` с кодеком `hevc` (можно запросить у дизайнера нужные форматы)

```
'video/mp4; codecs="hvc1"' // type в <source />
```

**Особенность chrome:** он не воспринимает кодек `hvc1` <br />
_Лучшее решение_ - дать браузеру самостоятельно определить кодек (то есть не указывать его вообще)

```
'video/mp4' // type в <source />
```

Особенность windows / android: некорректно отображает `mp4` - отображает прозрачность как черный фон <br />
_Решение_: переход на webp с определенным кодеком vp9 (в других не поддерживается прозрачность)

```
'video/webm; codecs=vp09.00.41.08' // type в <source />
```

**В компоненте** это решение реализовано с помощью атрибута `isSafari`.
Для Safari он будет отображать изначально `mp4` как основное видео:

```tsx
if (isSafari) {
  return (
    <>
      {Mp4Nodes}
      {WebmNodes}
    </>
  );
}
```

Для остальных систем windows, android и т.д приоритетом будет `webm`:

```tsx
return (
  <>
    {WebmNodes}
    {Mp4Nodes}
  </>
);
```

> **Реализация:** Используйте из шаблона `checkIsSafari` на стороне клиента
> или `checkIsSafariAgent` на стороне сервера, что бы определить - является ли устройство Safari или нет

## Превью на видео (poster)

Для компонента `Video` используется превью и стандартная кнопка проигрывания. Для обоих элементов используется компонент `Picture`, чтобы иметь возможность манипулировать изображениями.

## Загрузка видео

В шаблоне есть хук `useMediaReady` - сделан он для удобного управления готовностью видео / аудио.

Особенность этого процесса в том, что события для проверки готовности видео не вызываются при кешированном видео. В кешированном видео устанавливается сразу `readyState`, поэтому использовать только события - **не получится** - нужно проверять готовность состояния видео, что и происходит в этом хуке:

```tsx
if (video.readyState >= HTMLMediaElement.HAVE_FUTURE_DATA) {
  setMediaReady(true);
}
```

> В дополнении можно поиграться и совмещать различные состояния + события:
> Например на практике было, когда использовалось состояние `HAVE_METADATA` для проверки возможности запуска видео, но в хуке используется `HAVE_FUTURE_DATA` для надежности

## Поддержка

Рекомендуемый набор, который поддерживается на большинстве устройств:

1. формат mp4 + кодек AVC.
2. Профиль baseline/main/high <= 4.1

| Формат + Кодек                    | Итоговый Вердикт                                                                                | Объяснение                                                                                                                                                                                                                                                                                                                                              |
| --------------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **WebM + VP8+ (VP8/VP9)**         | Работает на всех современных платформах, но на macOS/iOS **альфа не отображается** корректно    | Safari (включая Chrome на iOS) не поддерживает альфа-дорожку. Видео воспроизводится, но прозрачность заменяется на чёрный фон.                                                                                                                                                                                                                          |
| **WebM + AV1**                    | Работает на новых устройствах с поддержкой AV1; на старых iOS/macOS воспроизведение не работает | AV1 поддерживает альфу теоретически. На новых Mac/iPhone с аппаратным декодером воспроизводится корректно. На старых iOS/macOS видео **не загружается**. На Android и Windows AV1 **поддерживается в современных браузерах** (Chrome, Edge, Firefox), воспроизводится без проблем; на старых устройствах может потребоваться программное декодирование. |
| **MP4 + H.264 (AVC)**             | Работает везде, но **без альфы**                                                                | Универсальный формат; H.264 не поддерживает прозрачность.                                                                                                                                                                                                                                                                                               |
| **MP4 + HEVC (H.265) с альфой**   | Работает на iOS/macOS с прозрачностью; **не работает** на большинстве других платформ           | Apple-стек (AVPlayer, QuickTime) умеет рендерить HEVC+alpha; остальные платформы не поддерживают или сильно зависят от ОС/лицензий.                                                                                                                                                                                                                     |
| **MOV (QuickTime, любые кодеки)** | Надёжно работает на iOS/macOS; **не поддерживается** на Android/Windows                         | MOV — контейнер Apple, поддерживает разные кодеки (H.264, ProRes, HEVC). Вне экосистемы Apple поддержка ограничена.                                                                                                                                                                                                                                     |
| **HLS (MP4/HEVC сегменты)**       | Отлично работает на iOS/macOS; ограниченно — на других платформах                               | HLS поток нативно поддерживается на iOS. Прозрачность применяется редко.                                                                                                                                                                                                                                                                                |
| **Ogg + Theora**                  | Поддержка ограничена; редко используется                                                        | Практически устаревший формат.                                                                                                                                                                                                                                                                                                                          |

## Финал

- Для Safari (iOS/macOS) — mp4 с HEVC (H.265) для альфы; fallback на AVC без прозрачности.
- Для остальных (Windows, Android, Chrome) — webm с VP9 для альфы; fallback на MP4.
- Определяем браузер через checkIsSafari/checkIsSafariAgent и меняем порядок <source> соответственно.
- Контролируем готовность видео через useMediaReady (readyState >= HAVE_FUTURE_DATA).
