# Настройка окружения для разработки

Перед началом работы с проектом необходимо настроить рабочее окружение. Этот документ содержит все необходимые шаги для подготовки к разработке.

## 🚀 Особенности проекта

В проекте используется **React Compiler**, который автоматически оптимизирует производительность компонентов. **Ручная мемоизация (useMemo, useCallback) не требуется** - компилятор позаботится об оптимизациях.

## ⚙️ Настройка VS Code

Для обеспечения корректной работы и соблюдения code style установите следующие расширения:

### Обязательные расширения:
- **ESLint** - анализ и проверка JavaScript/TypeScript кода
- **Prettier - Code formatter** - автоматическое форматирование кода
- **Stylelint** - линтинг CSS/SCSS стилей

# Настройка приватного реестра npm для пакетов компании

## 1. Регистрация Scope на компьютере

Откройте консоль и выполните команду:

```bash
npm config set @chulakov:registry https://gitlab.chulakov.org/api/v4/projects/236/packages/npm/
```

__Что это делает ?__

- Создает связь между scope @chulakov и вашим приватным реестром GitLab

- Все пакеты, начинающиеся с @chulakov/,  будут автоматически загружаться с указанного адреса

- Стандартные пакеты npm по-прежнему берутся с глобального реестра npmjs.org

## 2. Настройка авторизации

В консоли выполните команду:

```bash
npm config set //gitlab.chulakov.org/api/v4/projects/236/packages/npm/:_authToken=YOUR_TOKEN
```

- YOUR_TOKEN замените на свой токен доступа

__Как создать токен ?__

1) Заходим в GitLab

2) Нажимаем на свою авайтарку

3) Нажимаем на Preferences

4) Нажимаем на Access tokens

5) Добавляем новый токен (Add new token)

6) Из предоставленных разрешений выбираем все для чтения (read_...)

7) Копируем его и вставляем вместо YOUR_TOKEN

## 3. Использование пакетов

После настройки вы можете использовать пакеты компании. Например:

```bash
npx @chulakov/chulakov-next-starter
```
